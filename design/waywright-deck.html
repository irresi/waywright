The problem

Agents ship code faster
than teams can follow.

Agent side

Decision amnesia

Every session forgets why the last architecture was chosen, what was already rejected, and why the previous attempt failed. It re-derives from zero.

Human side

Cognitive debt

A green build closes the machine loop. But the developer never learns why the change was made — understanding falls behind output.

WAYWRIGHT02
The idea
One shared engineering memory.
The agent reads it to decide what to build next.
The human reads it to understand what shipped.

Most agents execute a direction you hand them. Waywright finds the direction from memory + prior art, ships it, and writes the outcome back — so the next decision stands on the last one.

How it works

A loop that closes on itself.

recall memory→ scout prior art→ decide→ implement→ CI observe→ self-correct
policy gate→ merge→ write outcome back to memory ↻ next run recalls it

Navigator finds the way · Actuator drives it · Buildkite is the eyes · Pomerium is the boundary · memory makes it cumulative.

WAYWRIGHT06
Repaying cognitive debt

Every run reconstructs its own decision.

A self-contained run record shows the human what the agent saw: recalled memory, candidates it weighed, why it rejected the others, the prior-art it cited, the Buildkite→Pomerium→merge timeline, and what it learned for next time.

Not a dashboard

It doesn't show "build is green." It shows why this change exists — the reasoning a bare diff throws away.

The same record

Feeds the agent's next decision and the human's understanding. One source of truth, two readers.
