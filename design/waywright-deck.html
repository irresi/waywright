The problem

Agents ship code faster
than teams can follow.

Agent side

Bounded by the model

The agent only knows what its model knows. When a task needs an ability the model lacks, it stalls or guesses instead of reaching for a proven tool.

Human side

Cognitive debt

A green build closes the machine loop. But the developer never learns why the change was made — understanding falls behind output.

WAYWRIGHT02
The idea
Don't generate from scratch.
Reuse what already works, and buy the ability you lack.

Most agents execute a direction you hand them. Waywright finds the direction — scouting prior art instead of re-deriving, and acquiring a paid capability when the model's own reach runs out — then drives it all the way to a policy-gated merge.

WAYWRIGHT03
How it works

A loop that closes on itself.

goal→ scout prior art→ acquire capability→ implement→ CI observe→ self-correct
policy gate→ merge→ teach the human

Buildkite is the eyes on CI · Pomerium is the boundary on merge · Zero.xyz is the reach past the model's limits.

WAYWRIGHT05
The proof · one closed loop, all real

Vague goal → real merge, no human in the loop.

What it found & fixed

issue #1 → PR #2, squash-merged: the exec log counted UTF-16 code units, not codepoints, so multi-byte text was mislabeled. A linter it bought from Zero surfaced the class of bug.

How it stayed safe

Bounded correction budget, minimal code-only diffs, CI green before any merge request, and a fail-closed Pomerium gate on the one irreversible step.

Every candidate, rejected alternative, build, and correction is a durable artifact — github.com/irresi/openclaw/pull/2

WAYWRIGHT06
Repaying cognitive debt

Done is when the human understands.

A merge doesn't end the loop. Waywright turns the merged change into a human-context artifact that shows why the change exists — recalled context, the alternatives it rejected, the prior art it cited, and the Buildkite→Pomerium→merge timeline.

Not a dashboard

It doesn't just say "build is green." It shows the reasoning a bare diff throws away.

One record, two readers

The same artifact feeds the agent's next decision and the human's understanding.
